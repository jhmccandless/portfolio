Death in the Afternoon

This application is designed as a point-of-sale system with a set group of cocktails imported from an API. One chooses one of the six drinks and is them redirected to the drink informatin page. After seeing the drink details and confirming this is the desired drink, you can add the drink and go to the cart page where the order is totaled. As a group project, jobs were divied out from person to person with my main job focusing on the second, drink detail page. I also had a large responsibility in the way the application visually looks. Other, smaller jobs included helping the team with problem solving issues as well as leading meetings to be sure tasks were handled and completed. We used not only the fundamental Javascript, CSS, and HTML, but also axios to hanlde the API calls as well as local storage to pass the information form page to page as the user clicks through.

kCalorie

Since the world does not have enough tracking applications, here is another one that tracks calories one eats throughout the day. This application not only tracks the calories, but also gives some statistics about the calories including the amount consumed over a 7-day period and and average calories per day and per meal. The design and logic behind the application came from me as this project I worked alone on. We used Node.js and PostgreSQL to interact with the backend. Middleware made the login page functional by using Github credentials that are then stored and used to create a personalized dashboard and detail page with data from that one user's input. Not only were Node.js and PostgreSQL used in this application, but Javascript, HTML and CSS were used as well.
